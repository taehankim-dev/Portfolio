# 스마트팜 미니어처 제어 시스템

## 1. 프로젝트 요약
- 협업사 제작 스마트팜 미니어처를 실제로 동작시키기 위한 라즈베리파이/아두이노 기반 제어 시스템
- 자회사 박람회 전시 일정에 맞춰 미니어처 내 물 분사, 도어 개폐, LED, 환기 팬 등을 제어하는 코드 구현
- 제어 로직 전반 담당 (단독 개발)

## 2. 내가 맡은 역할
- Python 기반 하드웨어 제어 로직 구현
- 외부 시스템과의 간단한 연동을 위한 API 구성

## 3. 핵심 경험

- **하드웨어 제약을 고려한 제어 방식 전환 경험**
  초기에는 라즈베리파이를 중심으로 미니어처 제어 로직을 구성했으나,
  실제 구동 과정에서 모터 및 배선 등 하드웨어 특성으로 인해
  안정적인 동작이 어려운 상황을 경험.
  문제 원인이 소프트웨어가 아닌 하드웨어와의 궁합에 있음을 인지하고,
  미니어처 제작사에서 제공한 아두이노 기반 제어 방식으로 전환함.

- **라즈베리파이–아두이노 연동을 통한 안정적인 구동 구현**
  제작사에서 제공한 아두이노 관련 자료를 바탕으로
  미니어처 구동 로직을 재구성하고,
  라즈베리파이와 아두이노를 함께 사용하는 구조로 수정.
  제한된 일정 속에서 기존 계획을 고집하기보다
  실제 전시 환경에서 안정적으로 동작하는 방식을 우선하여
  반복 테스트와 빠른 수정으로 시스템을 완성함.

## 4. 사용 기술
- **Control Logic**: Python 3.11, FastAPI, Uvicorn
- **Hardware Interface**: Raspberry Pi GPIO (gpiozero), PySerial (Arduino), aiohttp (SSE)

## 5. 비고
- 박람회 일정에 맞춰 빠른 동작 구현을 우선으로 진행한 전시용 시스템

경험에 대한 면접 대응 답변

“처음에는 라즈베리파이로 직접 모터를 제어하려고 했는데,
실제 구동 과정에서 상단 개폐 모터가 안정적으로 동작하지 않았습니다.

이때 소프트웨어 로직보다는
모터 구동에 필요한 전류나 신호 제어 방식의 차이 문제일 수 있다고 판단했고,
미니어처 제작사에서 제공한 아두이노 기반 제어 방식을 적용해봤습니다.

그 결과 상단 개폐는 아두이노를 통해서는 정상적으로 동작했지만,
측면 비닐 한 쪽은 아두이노로 연결해도 동작하지 않아
해당 부분은 모터나 배선 자체의 문제로 판단했습니다.

이 경험을 통해 컨트롤러의 문제가 아니라
하드웨어 자체 문제일 수도 있다는 걸 구분해서 보게 되었고,
일정이 촉박한 상황에서는
원인을 끝까지 파기보다는 ‘정상 동작하는 경로를 선택하는 판단’을 했습니다.”
