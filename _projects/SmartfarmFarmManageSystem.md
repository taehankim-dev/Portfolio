# 제목

## 1. 프로젝트 한 줄 요약

- MQTT 기반 센서 데이터를 수집하여 농가 환경을 실시간으로 모니터링하는 스마트팜 관리 시스템

---

## 2. 프로젝트 배경

- 기존 회사에서 사용하던 스마트 공장 및 스마트 팩토리를 관리하던 사이트와는 별도로 스마트팜만을 관리하는 사이트의 필요성 대두.
- 사용자 친화적이며, 모바일 대응이 가능한 관리 사이트의 필요.

---

## 3. 나의 역할 (책임 범위)

- 프론트엔드 전반 설계 및 구현
- 백엔드 API 및 DB 구조 설계
- MQTT 기반 센서 데이터 수집·저장 및 웹 제어 API 연동 로직 설계
- Docker 기반 배포 환경 구성 및 운영

---

## 4. 핵심 기술 스택 (대표 3~5개)

- Frontend: Next.js, TypeScript
- Backend: Spring Boot
- DB: PostgreSQL
- Infra: Docker, Nginx
- ETC: MQTT

---

## 5. 기술적 판단 & 설계 포인트 ⭐ (가장 중요)

- 실시간 데이터 처리를 위해 MQTT 프로토콜을 채택
- 프론트엔드와 백엔드를 분리하여 역할과 책임을 명확히 하고 유지보수성 확보
- 웹 제어 API와 MQTT 데이터 수집용 백엔드를 분리하여 트래픽 간섭 및 장애 전파 방지
- Docker를 활용해 배포 환경을 표준화하고 운영 리스크 감소

---

## 6. 문제 상황 & 해결 경험

- 문제: 백엔드 재배포 시 알림이 중복 발송되는 문제 발생  
  - 원인: 메모리 기반 상태 관리로 인해 재기동 시 상태 초기화  
  - 해결: 알림 발송 상태를 DB에 저장하도록 구조 개선  
  - 결과: 문자 발송 비용 감소 및 알림 안정성 확보

- 문제: MQTT 데이터 수집 시 웹 제어 API 응답 지연 발생  
  - 원인: MQTT 데이터 수집 로직이 동기적으로 처리되어 웹 제어 API가 후순위로 밀림  
  - 해결: MQTT 데이터 수집 전용 백엔드를 분리하여 비동기 처리  
  - 결과: 웹 제어 API 응답 속도 개선 및 시스템 책임 분리

- 문제: 실시간 데이터가 아닌 과거 데이터가 우선 표시되는 현상  
  - 원인: MQTT retain 설정으로 인해 이전 데이터가 우선 수신  
  - 해결: 일정 시간 이상 차이가 나는 데이터는 비동기 저장 처리  
  - 결과: 실시간 데이터 정상 표시

- 향후 앱 형태로 확장 시 사용자 경험 개선 가능성 인식

# 이력서용 요약

스마트팜 관리 시스템 개발

- MQTT 기반 센서 데이터를 수집·저장하여 농가 환경을 실시간으로 모니터링하는 스마트팜 관리 시스템 설계 및 구현
- 프론트엔드(Next.js)와 백엔드(Spring Boot) 분리 구조로 웹 제어 API와 데이터 수집 로직을 분리해 안정성 확보
- 운영 중 발생한 알림 중복 발송, 실시간 데이터 처리 문제를 구조 개선을 통해 해결
- Docker 기반 배포 환경 구성으로 실제 스마트팜 운영 환경에 안정적으로 적용
