> **프로젝트 개요**
>
> 스프레드시트 기반 수작업 견적으로 인한 계산 오류와 시간 소요를 해결하기 위한 프로젝트입니다. 영업 현장에서 즉시 예상 견적을 제공할 수 있는 웹 기반 시스템을 단독 개발했으며, Capacitor를 활용해 하이브리드 앱으로 확장했습니다.
>
> **프로젝트 성격**: 영업 현장의 비효율을 시스템으로 해결한 실무형 견적·상담 시스템

---

## 문제 상황 & 해결 경험

**문제**: 스프레드시트 기반 수작업 견적으로 인한 계산 오류 및 시간 소요, 배포 후 이메일 전송 실패 및 페이지 로딩 지연

**역할**: 풀스택 개발 / 하이브리드 앱 전환 / 단독 개발

**결과**:
- 렌더링 속도 10초 이상 → 5초 이하로 개선
- 영업 현장에서 실사용
- 견적 산출 오류 감소

---

## 문제 상황 & 해결

### 1. 이메일 이미지 용량 초과

**현상**: 예상 견적 결과를 이메일로 전달하는 과정에서 이미지 용량 초과로 발송 거부

**원인**:
- 견적 결과 화면을 그대로 캡처하여 이메일 첨부
- 이메일 서비스의 이미지 용량 제한 초과

**해결**:
- 레이아웃 단순화: 불필요한 요소 제거
- 핵심 정보 위주 재구성: 평수, 총 견적 금액, 주요 자재 항목만 포함
- 이미지 압축: 캡처 시 해상도 조정

**결과**:
- 이메일 발송 실패 문제 해소
- 영업 현장에서 견적 전달 안정성 확보

### 2. 페이지 로딩 속도 지연

**현상**: 견적 계산 결과 페이지의 초기 로딩 속도가 느려 현장 상담 흐름이 끊김

**원인**:
- 단일 컴포넌트에서 모든 계산 로직을 처리
- 계산 완료 전까지 화면이 렌더링되지 않음

**해결**:

컴포넌트를 섹션별로 분리하여 각 영역이 독립적으로 계산·렌더링되도록 개선했습니다.

```vue
<!-- Before: 단일 컴포넌트에서 모든 계산 -->
<template>
  <div v-if="!loading">
    <!-- 모든 견적 항목 -->
  </div>
</template>

<script setup>
const loading = ref(true);

onMounted(async () => {
  // 모든 계산을 순차적으로 처리
  await calculateMaterials();
  await calculateLabor();
  await calculateProfit();
  loading.value = false;
});
</script>
```

```vue
<!-- After: 컴포넌트 분리 및 개별 렌더링 -->
<template>
  <div>
    <MaterialSection />  <!-- 자재 계산 및 렌더링 -->
    <LaborSection />     <!-- 인건비 계산 및 렌더링 -->
    <ProfitSection />    <!-- 이익률 계산 및 렌더링 -->
  </div>
</template>

<script setup lang="ts">
// 각 섹션 컴포넌트에서 독립적으로 계산 및 렌더링
</script>
```

**결과**:
- 페이지 렌더링 속도: **10초 이상 → 5초 이하**로 개선
- 상담 중 대기 시간 감소
- 사용자 UX 향상
