> **프로젝트 개요**
>
> 웹 기반 MES(Manufacturing Execution System) 프론트엔드 개발 프로젝트입니다. Vuetify를 활용한 UI 퍼블리싱과 데이터 시각화 기능을 구현했으며, Atomic Design 개념을 참고하여 재사용 가능한 컴포넌트 구조를 설계했습니다.

> **프로젝트 성격**: 체계적인 컴포넌트 설계를 통해 유지보수성과 재사용성을 확보한 UI 개발 프로젝트

## 기술적 판단 & 설계 포인트

### Atomic Design 도입 — 선택 이유와 실제 적용 기준

MES 시스템은 화면 수가 많고 유사한 UI 패턴이 반복되는 특성이 있다. 입출고 화면, 생산 현황 화면, 불량 관리 화면이 모두 DataTable + SearchBar + StatusBadge 조합을 사용했는데, 이를 각 화면에 별도로 구현하면 디자인 수정 한 건이 전체 화면을 순회하는 작업이 된다.

**Atomic Design을 선택한 이유**: 변경이 전파되는 범위를 계층으로 통제할 수 있다. Button 색상을 바꾸면 Button을 사용하는 모든 Molecule·Organism이 자동으로 반영된다. 반대로 SearchBar 레이아웃 변경은 SearchBar를 사용하는 화면에만 영향을 주고 Button에는 영향을 주지 않는다.

**컴포넌트 구조**:
```
src/components/
├── atoms/           # 단독으로 의미를 갖는 최소 단위 (변경 시 전체 전파)
│   ├── Button.vue
│   ├── Input.vue
│   └── Badge.vue
├── molecules/       # Atom의 조합 — 단일 기능 단위
│   ├── SearchBar.vue
│   ├── FormField.vue
│   └── StatusCard.vue
├── organisms/       # 화면의 독립적인 구역 단위
│   ├── DataTable.vue
│   ├── ProductionChart.vue
│   └── Header.vue
└── templates/       # 페이지 레이아웃 골격만 담당
    ├── DashboardLayout.vue
    └── FormLayout.vue
```

**실제 적용 판단 기준**: 처음에는 모든 요소를 Atomic Design 계층에 맞추려 했지만, 화면 전용 UI는 굳이 atoms/molecules로 분리하면 오히려 복잡도만 높아졌다. 재사용이 2회 이상 확인된 경우에만 상위 계층으로 분리하는 실용적 기준을 세웠다.

**재사용 가능한 SearchBar 컴포넌트 예시**:

```vue
<template>
  <div class="search-bar">
    <v-text-field
      v-model="searchQuery"
      placeholder="검색..."
      prepend-inner-icon="mdi-magnify"
      variant="outlined"
      density="compact"
      @keyup.enter="handleSearch"
    />
    <v-btn color="primary" @click="handleSearch">검색</v-btn>
  </div>
</template>

<script setup lang="ts">
import { ref } from 'vue';

const searchQuery = ref('');
const emit = defineEmits<{ search: [query: string] }>();
const handleSearch = () => emit('search', searchQuery.value);
</script>
```

이 컴포넌트는 `search` 이벤트만 emit하고 실제 조회 로직은 부모 컴포넌트가 담당한다. UI 컴포넌트가 비즈니스 로직을 모르게 설계한 덕분에, 같은 SearchBar를 입출고·생산·불량 화면 모두에서 재사용하면서 각 화면의 조회 로직은 독립적으로 유지할 수 있었다. 이 "UI와 로직의 분리" 원칙은 이후 백엔드 설계에서 레이어 분리 개념으로 자연스럽게 이어졌다.

## 결과

### 구조적 성과
- **변경 전파 범위 통제**: Atomic Design 계층 구조로 공통 컴포넌트 수정이 사용처 전체에 일관되게 반영 — 화면별 개별 수정 불필요
- **재사용 기반 확보**: SearchBar, DataTable, StatusCard 등 약 30개 공통 컴포넌트 구축 — 유사 화면 추가 시 조합만으로 구현
- **설계 관점 전환**: UI 컴포넌트에서 로직을 분리하는 과정에서 "책임 분리"와 "경계 설계"의 중요성을 체감 — 이후 백엔드 레이어 설계로 이어진 관심의 시작점
