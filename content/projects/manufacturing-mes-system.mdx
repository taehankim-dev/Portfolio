## 문제 상황 & 해결 경험

**문제**: 수기로 관리되던 입출고 및 재고 관리의 작성 누락과 자료 관리 어려움, 배포 후 사용자의 업무 흐름에 맞지 않는 UI로 인한 효율 저하

**역할**: 프론트엔드 주요 기능 구현 / 백엔드 일부 API 개발 / Nginx 배포 환경 구성

**결과**:
- 사용자 피드백 반영으로 데이터 조회 편의성 향상
- 엑셀 업로드 전처리 로직 추가로 입력 안정성 확보
- 실제 기업에서 안정적으로 사용 중

---

## 배경

재입사 후 첫 프로젝트로, 중소 제조기업의 자재·제품 입출고 및 재고 관리를 전산화하는 시스템 개발이 필요했습니다.

기존 상황:
- **수기 관리의 한계**: 입출고 관리와 재고 관리를 종이로 진행하면서 작성 누락 및 오기재 빈번
- **자료 추적 어려움**: 과거 데이터 검색이 어렵고, 재고 현황 파악에 시간 소요
- **전산화 필요성**: 업무 효율 개선과 데이터 정확도 향상을 위한 시스템 도입

---

## 나의 역할

2인 개발 환경에서 다음 역할을 담당했습니다:

- **프론트엔드 주요 기능 개발**: Vue 3와 Ag-grid 기반 입출고 관리 화면 구현
- **백엔드 일부 API 구현**: 데이터 조회 및 엑셀 업로드 API 개발
- **배포 환경 구성**: Nginx 기반 배포 환경 설정 및 운영
- **사용자 소통**: 실제 사용자와 직접 소통하며 피드백 수렴 및 기능 개선

## 문제 상황

### 1. 업무 흐름에 맞지 않는 UI

**현상**: 배포 초기, 입출고·생산·출고 데이터를 한 화면에서 확인하기 어려워 업무 효율 저하

**원인**:
- 개발자 관점에서 설계한 UI가 실제 업무 흐름과 불일치
- 조회 기준(일별/월별/연간)이 복잡하고 직관적이지 않음

**해결 과정**:
1. **사용자와 직접 소통**: 실제 업무 프로세스와 데이터 조회 패턴 파악
2. **조회 기준 단순화**: 복잡한 조회 옵션을 일별/월별/연간으로 단순화
3. **화면 분리**: 연관된 데이터를 그룹화하여 화면을 명확히 분리
4. **대안 제시**: 기술적 제약 사항은 사용자에게 설명하고 대안 제시

**결과**:
- 데이터 조회 편의성 향상
- 사용자 이해도 및 만족도 증가

### 2. 엑셀 업로드 오류

**현상**: 엑셀 파일 업로드 시 사용자 입력 실수로 인한 오류 발생

**원인**:
- 사용자가 엑셀 파일에 공백 행을 포함하여 업로드
- 백엔드에서 공백 행을 데이터로 인식하여 처리 오류 발생

**해결**:
```java
@PostMapping("/upload/excel")
public ResponseEntity<?> uploadExcel(@RequestParam("file") MultipartFile file) {
    List<RowData> rows = excelParser.parse(file);
    
    // 공백 행 필터링
    List<RowData> validRows = rows.stream()
        .filter(row -> !row.isEmpty())  // 공백 행 제외
        .filter(row -> row.hasRequiredFields())  // 필수 필드 검증
        .collect(Collectors.toList());
    
    dataService.saveAll(validRows);
    return ResponseEntity.ok("업로드 완료");
}
```

**결과**:
- 엑셀 업로드 기능 안정화
- 사용자 입력 실수에 대한 내결함성 확보

## 기술적 판단 & 설계 포인트

### 1. 기존 시스템 스타일 분석 우선

재입사 후 첫 프로젝트로, 새로운 구조 도입보다는 **기존 회사 시스템 및 개발 스타일을 분석**하여 일관성을 유지하는 방향으로 설계했습니다.

- 회사 내 다른 시스템과 유사한 UI/UX 패턴 사용
- 기존 코드 컨벤션 및 폴더 구조 준수
- 팀원과의 협업을 위한 코드 가독성 우선

### 2. 안정성 우선 설계

초기 도입 기업 환경을 고려하여 기능 확장보다는 **핵심 기능의 정확성과 사용성**을 우선했습니다.

- 복잡한 기능보다 필수 기능의 안정성 확보
- 사용자 적응 비용 최소화
- 점진적 개선 가능한 구조
